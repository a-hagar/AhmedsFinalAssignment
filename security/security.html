Security

This site has security headers to protect users from potential risks such as:

  • Cross-site scripting
  • Sensitive Data Exposure
  • Broken Access Control
  • Security Misconfigurations

This site does not collect information from users so there is no issues with handling sensitive data or insecure deserialization. This site does not require a user login, and avoids any potential Broken Authethication. By not being connected to a database, SQL injections will not impact this site.
